Cursor 额度怎么看？
实时查询余额与用量详解

担心写代码写到一半突然“断粮”？本文教你 2 招快速查看 Cursor 剩余额度，合理规划 AI 资源。

2026-01-22
Cursor-VIP Team

为什么需要关注额度？
Cursor Pro 用户每月拥有 500 次快速请求 (Fast Requests) 额度。一旦用完，虽然还能继续使用，但会进入“慢速队列”，响应速度可能变慢，且无法使用 Claude 3.5 Opus 等高级模型的 Max 模式。及时查看额度，能让你心中有数。

方法一：App 内实时显示 (最推荐)

这是最方便快捷的方法，只需设置一次，之后在每次对话时都能直观看到剩余百分比。

Step 1

打开 Cursor App，点击右上角齿轮图标或使用快捷键 Cmd/Ctrl + Shift + J 进入 Settings 。

Step 2

在设置页面的搜索框中输入 usage summary 。

Step 3

找到 "When to show the usage summary" 选项，将下拉菜单设置为 Always 。

设置完成后，当你打开 Chat 聊天界面时，输入框下方就会常驻显示一行小字，例如 "14% of monthly fast requests used" ，一目了然。

方法二：网页版 Dashboard (更详细)

如果你想查看具体的账单周期、每个模型的具体消耗次数，或者导出数据，网页版是更好的选择。

  • 1
    访问官网

    在浏览器访问 Cursor Settings 并登录你的账号。

  • 2
    查看 Usage 面板

    在左侧菜单点击 Usage 。这里会以图表形式展示你本月的 Fast Requests、Slow Requests 以及 Copilot++ 的代码补全行数。

额度用完了怎么办？

当你把 500 次快速额度用完后，不必惊慌，你有以下几种选择：

进入慢速队列 (免费)

你可以继续无限次使用标准模型，但可能会遇到排队等待。如果你不赶时间，这是最经济的选择。

开启按量付费 (On-Demand)

在设置中开启 "Usage-based pricing"，超出部分按次计费（约 $0.04/次）。适合临时急需的情况。建议在 Spending 页面设置上限，避免账单爆炸。

使用自定义 API Key

在模型设置中填入你自己的 OpenAI 或 Anthropic API Key，这样消耗的就是你 API Key 的额度，不再占用 Cursor 的额度。
